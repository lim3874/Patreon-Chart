# v2.9.0 - Expanded Patreon API Export

## Added

- The Patreon API sync now requests every documented member field available from the campaign members endpoint.
- The export now includes expanded user, tier, campaign, pledge history, and optional address data.
- Added tier Discord role IDs, campaign Discord server ID, pledge history summaries, raw pledge history JSON, raw social connection JSON, Patreon profile metadata, and campaign metadata to the Patreon API CSV.
- Added additional Patreon API table columns for the expanded fields.
- Added tests for expanded Patreon member row parsing.

## Changed

- Patreon member import falls back cleanly when optional address data is unavailable for the current OAuth token.
- Patreon API table sorting now handles expanded amount, count, date, and boolean fields using typed values.

## Notes

- Discord names are not included in Patreon API responses. Patreon provides the Discord user ID through `social_connections.discord.user_id`; resolving that ID into a Discord username or server nickname requires a Discord bot token and the target Discord server ID.

## Verification

- `python -m py_compile patreon_api.py patreon_member_gui.py export_patreon_members.py`
- `python -m unittest discover -s tests`
- Live Patreon API smoke test fetched 472 members with 106 exported CSV fields.
