# v2.10.0 - Discord Bot Member Resolution

## Added

- Added optional Discord bot integration for resolving Patreon Discord user IDs into Discord usernames, display names, server nicknames, role names, and server join dates.
- Added `Discord 봇 설정` in the Patreon API workspace. The bot token and server ID are stored locally in `discord_credentials.json`, which is ignored by Git.
- Added `Discord 정보 채우기` so existing Patreon API CSV rows can be enriched without reloading Patreon.
- Added Discord lookup fields to the Patreon API CSV and member table:
  - lookup status
  - display name
  - global name
  - server nickname
  - role IDs and role names
  - server join date
  - Nitro boost date when available
  - raw member JSON
- Added a test for flattening Discord guild member API responses.

## Changed

- Patreon current-member imports now automatically attempt Discord enrichment when Discord credentials are configured.
- If Discord lookup fails during a Patreon import, the Patreon import still completes and writes the Patreon data; the Discord error details are written to `output/discord_api_error.log`.

## Notes

- Discord usernames and server nicknames require a Discord bot token and the target server ID. Patreon itself only provides `social_connections.discord.user_id`.
- If a bot token was pasted into chat or otherwise exposed, reset it in Discord Developer Portal before saving it in the app.

## Verification

- `python -m py_compile .\discord_api.py .\patreon_api.py .\patreon_member_gui.py .\export_patreon_members.py`
- `python -m unittest discover -s tests`
